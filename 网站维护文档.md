## 1. 项目概述

本项目是 **西安建筑科技大学（XAUAT）ACM 协会官方网站**，基于 VitePress 构建的静态站点，托管于 GitHub Pages。

### 网站主要功能

- **首页**：文章列表 + 分页，打字机标语动画，全宽 Hero 区域
- **文章系统**：支持分类、标签、分页的博客文章
- **关于我们**：协会介绍、赛事科普（ICPC / CCPC / 蓝桥杯 / 天梯赛）、活动日历、入门指南
- **集训队**：成员展示、获奖荣誉（选项卡切换）、获奖瞬间（照片墙）
- **互动**：Giscus 评论区、页面访问统计、友链
- **其他**：文章归档、标签筛选、404 页面、深色模式

### 仓库信息

- **GitHub 仓库**：`xauat-acm/xauat-acm.github.io`
- **主分支**：`main`
- **部署目标**：GitHub Pages

## 2. 技术栈

| 类别 | 技术 | 版本 |

| 静态站点生成器 | VitePress | ^1.6.3 |

| 服务 | 加载方式 | 用途 | 失败影响 |

## 4. 本地开发指南

### 4.1 环境要求

- **Node.js**：>= 18（推荐 20 LTS）
- **npm**：>= 9
- **Git**：最新稳定版

### 4.2 首次搭建

```bash
# 1. 克隆仓库
git clone https://github.com/xauat-acm/xauat-acm.github.io.git
cd xauat-acm.github.io

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

开发服务器默认运行在 `http://localhost:5173`，支持热重载。


### 4.3 常用命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动本地开发服务器（热重载） |
| `npm run build` | 生产构建，输出到 `docs/.vitepress/dist/` |
| `npm run preview` | 本地预览构建产物 |

### 4.4 开发工作流

1. 在本地创建功能分支：`git checkout -b feature/xxx`
2. 运行 `npm run dev` 启动开发服务器
3. 编辑文件，浏览器自动刷新
4. 完成后提交并推送：`git push origin feature/xxx`
5. 在 GitHub 上创建 Pull Request 合并到 `main`
6. 合并后自动触发部署

## 5. 内容管理

### 5.1 添加新文章

在 `docs/posts/` 目录下创建 `.md` 文件，文件名建议使用英文短横线命名：

```markdown
---
title: 文章标题
date: 2026-06-22
category: 算法入门
tags:
  - 动态规划
  - 背包问题
excerpt: 这是一篇关于动态规划的入门教程。
---

## 正文内容

在这里撰写文章正文...
```

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 文章标题 |
| `date` | ✅ | 发布日期（YYYY-MM-DD） |
| `category` | ❌ | 分类名称 |
| `tags` | ❌ | 标签列表（YAML 数组） |
| `excerpt` | ❌ | 摘要，不填则自动截取正文前若干字 |

### 5.2 编辑页面内容

页面文件都是标准 Markdown，可直接编辑：

| 页面 | 文件路径 | 编辑内容 |
|------|----------|----------|
| 关于我们 | `docs/about.md` | 协会简介、联系方式 |
| 赛事介绍 | `docs/about/competitions.md` | 赛事对比表 |
| 协会活动 | `docs/about/activities.md` | 年度活动日历 |
| 入门指南 | `docs/about/guide.md` | 入门路线、FAQ |
| 集训队 | `docs/team.md` | 成员信息表格 |
| 获奖荣誉 | `docs/award.md` | 获奖记录 |
| 友链 | `docs/links.md` | 友情链接 |

### 5.3 更新导航菜单

编辑 `docs/.vitepress/config.mts` 中的 `nav` 配置：

```typescript
nav: [
  { text: '首页', link: '/' },
  {
    text: '集训队',
    items: [
      { text: '成员', link: '/team' },
      { text: '获奖荣誉', link: '/award' },
      { text: '获奖瞬间', link: '/gallery' },
    ],
  },
  // ... 在此添加更多导航项
],
```

### 5.4 文章数据加载机制

文章数据通过 `docs/posts.data.mts` 自动扫描 `docs/posts/` 目录，读取每篇文章的 frontmatter，按日期降序排序。数据通过 `composables/usePosts.ts` 实现跨页面缓存共享。

**添加文章后无需手动刷新数据**，开发服务器会自动重新加载。

## 6. 主题与样式定制

### 6.1 配色方案

配色定义在 `docs/.vitepress/theme/styles/custom.css` 中，通过 CSS 变量控制：

```css
:root {
  --vp-c-brand: #49b1f5;        /* 品牌主色（蓝色） */
  --vp-c-brand-light: #6ec1f7;  /* 亮色变体 */
  --vp-c-brand-dark: #1a93e0;   /* 暗色变体 */
  --vp-c-bg: #f6f8fa;           /* 页面背景 */
}
```

### 6.2 修改主题色

1. 打开 `docs/.vitepress/theme/styles/custom.css`
2. 找到 `--vp-c-brand` 变量
3. 修改颜色值，同时调整 `--vp-c-brand-light` 和 `--vp-c-brand-dark`
4. 搜索文件中所有 `#49b1f5` 并替换为新颜色（部分组件硬编码了颜色）

### 6.3 深色模式

深色模式通过 `html.dark` class 触发，VitePress 自动处理切换。自定义深色样式位于同文件的暗色区块：

```css
html.dark {
  --vp-c-bg: #0f0f1a;
  /* 其他暗色变量 */
}
```

### 6.4 布局结构

网站使用**自定义双栏布局**（覆盖 VitePress 默认布局）：

- **左侧（75%）**：主要内容区
- **右侧（25%，固定 320px）**：侧边栏（站点信息、公告、最新文章、分类、标签云）
- **文章页**：居中单栏（无侧边栏）
- **响应式断点**：1024px 切换为单栏，768px 调整字体和间距

修改布局需编辑：
- `docs/.vitepress/theme/Layout.vue` — 布局结构
- `docs/.vitepress/theme/styles/custom.css` — 布局样式

## 9. 部署流程

### 9.1 自动部署（推荐）

网站通过 GitHub Actions 自动部署。只需将代码推送到 `main` 分支：

```bash
git add .
git commit -m "更新内容"
git push origin main
```

推送后，GitHub Actions 会自动执行：
1. 检出代码
2. 安装 Node.js 20
3. 安装依赖（`npm ci`）
4. 构建网站（`npm run build`）
5. 部署到 GitHub Pages

约 1-2 分钟后网站更新生效。

### 9.2 手动部署

如果自动部署失败，可以手动构建并部署：

```bash
# 本地构建
npm run build

# 构建产物在 docs/.vitepress/dist/
# 将其内容部署到 GitHub Pages
```

### 9.3 部署配置 (`deploy.yml`)

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main, master]
  workflow_dispatch:  # 允许手动触发

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist
      - uses: actions/deploy-pages@v4
```

### 9.4 检查部署状态

1. 打开 GitHub 仓库页面
2. 点击 **Actions** 选项卡
3. 查看最新的工作流运行状态
4. 绿色 ✅ 表示部署成功

### 11.8 添加线下比赛

线下比赛显示在首页「即将举行的比赛（线下）」卡片中，数据存储在 `docs/public/offline-contests.json`。

**数据格式：**

```json
[
  {
    "name": "ICPC 亚洲区域赛（西安站）",
    "date": "2026-11-15",
    "location": "西安",
    "url": "https://icpc.global",
    "status": "upcoming"
  }
]
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 比赛名称（必填） |
| `date` | string | 比赛日期，格式 `YYYY-MM-DD`（必填） |
| `location` | string | 比赛地点，如「西安」「线上」（必填） |
| `url` | string | 比赛官网链接，没有可留空 `""` |
| `status` | string | 状态：`upcoming`（即将开始）/ `ongoing`（进行中）/ `past`（已结束） |

**操作步骤：**

1. 编辑 `docs/public/offline-contests.json`
2. 按 JSON 格式添加或修改比赛条目
3. 保存后本地 `npm run dev` 预览效果
4. 提交并推送，重新部署后生效

**排序：** 比赛按 JSON 数组顺序显示，建议将最近的比赛排在前面。

**删除过期比赛：** 比赛结束后，从 JSON 数组中删除对应条目即可。

## 12. 故障排查

### 12.1 本地开发问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| `npm run dev` 报错 | 依赖未安装或版本不兼容 | `rm -rf node_modules && npm install` |
| 页面 404 | 文件路径错误 | 检查 `.md` 文件是否在正确位置 |
| 样式不生效 | CSS 缓存 | 硬刷新浏览器（Ctrl+F5） |
| 热重载失效 | Vite 缓存问题 | 重启 `npm run dev` |

### 12.2 构建问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| `npm run build` 失败 | Markdown 语法错误 | 检查最近的 `.md` 编辑 |
| 构建产物不完整 | 磁盘空间不足 | 清理 `node_modules/.cache` |
| 构建后链接 404 | `base` 配置错误 | 检查 `config.mts` 中的 `base` 值 |

### 12.3 部署问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| Actions 运行失败 | 依赖安装失败 | 查看 Actions 日志定位错误 |
| 部署后页面空白 | `base` 路径不匹配 | 确认 `config.mts` 中 `base: '/'` |
| 静态资源 404 | 路径引用问题 | 检查图片引用是否以 `/` 开头 |

### 12.4 外部服务问题

| 问题 | 现象 | 处理方法 |
|------|------|----------|
| Giscus 不显示 | 评论区空白 | 检查仓库 Discussions 是否启用 |
| 访问统计不显示 | 统计数字为 0 或空白 | Busuanzi 服务波动，通常自行恢复 |
| canvas-ribbon 报错 | 控制台报错 | 不影响功能，可忽略或禁用该组件 |
